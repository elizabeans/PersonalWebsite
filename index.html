Projecteer is a web application for sharing projects. If someone is lacking a certain skillset or needs more manpower to bring their ideas to life, they can post their project on Projecteer to find more members. If someone is lacking a cool idea to work on and would like to join a team, they browse and join any project they're interested in. Projecteer is a way to find your "musketeers" and gain experience working on something they might have not been able to otherwise.

I created Projecteer for my final project at Origin Code Academy. I decided to deviate from the .NET stack we focused on during class to learn something new by adopting the MEAN stack. It was a challenge because I made the switch halfway through working on the final project. I liked the idea of being able to just use JavaScript throughout my code. This allowed me to further my knowledge in AngularJS and play around with new techniques like using a NoSQL database and running my project using Node. I also decided to try cookie-based authentication instead of token-based authentication like in previous projects.

Technology Used:
MEAN stack
Pusher API
HTML/CSS, Bootstrap and Angular-Material

Live demo @ http://projecteer.herokuapp.com or see the code on github

"It's Graze for Dogs!"

Pawze lets you setup a monthly subscription for a customizable box of your dog's favorite treats!

Pawze is the product of a two-week group project with a total of seven members. The main languages we used were C# and JavaScript. We built the web app using a domain driven design and a code first approach with Entity Framework. We created a SQL database using SQL Server. Following ASP.NET, we created our own API. We handled user creation and security using OWIN and ASP.NET Identity. For the front-end, we mainly focused on using AngularJS to create a RESTful architecture between the client and server. As for external resources, we used the Stripe API to handle our subscription services.

Technology Used:
API and Repository Layer built in C#
SQL Server
AngularJS
Bootstrap
Stripe API

Live demo @ http://pawzeweb.azurewebsites.net/ or see the code on github

Looking for something to do? Find events on campus using UCEvent!

UCEvent is the product of a 10-member group project in my CSE110 (Intro to Software Engineering) class at UCSD. We noticed that our campus lacked a central place where any upcoming events were posted and that students were largely unaware of most of the events being hosted on or around campus. We decided to create an Android app that facilitated that type of communication. We used Java and JavaScript to create our app. We hosted our data on Parse which was still available for the duration of our project.

The course was run like a mock company with our professor being the CEO. We followed agile methods and created sprints within our 10-week course. We would have weekly meetings with our CEO to document our progress. Instead of presenting the tasks we tackled, each member was given a role and had to create a presentation during their assigned week regarding the topic. For example, we had a team lead, tech lead, business analyst (in charge of user stories), QA team, architects, and UI designers.

Near the end of the quarter, we turned a promo video which determined which apps would be on top and we were chosen as one of the top 5.
